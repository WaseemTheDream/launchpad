# Launchpad - AI Agent Context

> This file provides context for AI agents working on this project.

## Project Overview

**Launchpad** is an AI-powered Android development platform designed to work seamlessly with Claude Code. It enables rapid ideation, prototyping, and iteration on Android applications.

### Purpose

- Help users go from idea → working Android app with AI assistance
- Provide structured workflows for product ideation and MVP generation
- Maintain context and documentation throughout the development process

### Tech Stack

- **Language**: Kotlin
- **UI Framework**: Jetpack Compose with Material Design 3
- **Build System**: Gradle 9.1.0 (Kotlin DSL)
- **Min SDK**: 24 | **Target SDK**: 36
- **Architecture**: Single-module app (expandable)

## Key Commands

### Product Development

| Command | Purpose |
|---------|---------|
| `/blueprint` | **Product Ideation** - Define app concept through interactive questionnaire. Works with or without existing product docs. Creates context files in `.claude/context/`. |
| `/forge` | **MVP Generation** - Generate MVP proposal, review with user, build complete working prototype. Uses Ultrathink for thorough analysis. |
| `/ship` | **Autonomous Changes** - Voice-friendly command to implement changes, document, and commit. Example: `/ship add dark mode toggle` |
| `/pilot` | **Interactive Development** - Hands-on development with user present for testing. Changes are NOT committed until `/land`. |
| `/land` | **Commit Pilot Changes** - Commit pending changes from a `/pilot` session with documentation. |
| `/flight-plan` | **Strategic Planning** - Deep brainstorming for complex features. Creates detailed implementation plans. |
| `/queue` | **Task Queue** - Save a task for later when AI is busy. Example: `/queue add error handling` |
| `/burn` | **Process Queue** - Execute all queued tasks like a rocket burn. |
| `/reboot` | **Resume Work** - Continue interrupted sessions from progress logs. |

### Build & Deploy

| Command | Purpose |
|---------|---------|
| `/build` | Compile the app (`gradlew assembleDebug`) |
| `/build clean` | Clean build from scratch |
| `/install` | Build and install to connected device/emulator |
| `/run` | Build, install, and launch the app |

### Automation

| Command | Purpose |
|---------|---------|
| `/execute-prompt <file>` | Execute a prompt file with logging |

## Conventions

### Commit Messages

All commits follow the pattern:
```
[NNNN] type: Brief description

- Detail 1
- Detail 2

Co-Authored-By: Claude <noreply@anthropic.com>
```

Types: `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`, `build`, `perf`

### Prompt Files

Prompt files in `.claude/prompts/` use 4-digit prefixes:
- `0001-feature-name.md`
- `0002-another-feature.md`

Directives supported:
- `[auto-commit]` - Automatically commit on success
- `[no-commit]` - Don't commit changes
- `[branch: name]` - Work on specific branch

### Session Documentation

Both `/blueprint` and `/forge` create session documentation prompts that record:
- All questions asked and answers given
- Decisions made
- Files created/modified
- Commits made

## Building the App

**Important**: Set JAVA_HOME before running Gradle commands:

```bash
export JAVA_HOME="/c/Program Files/Android/Android Studio/jbr"
```

Build commands:
```bash
# Build
export JAVA_HOME="/c/Program Files/Android/Android Studio/jbr" && "C:\Users\Waz\AndroidStudioProjects\launchpad\gradlew.bat" assembleDebug

# Install
export JAVA_HOME="/c/Program Files/Android/Android Studio/jbr" && "C:\Users\Waz\AndroidStudioProjects\launchpad\gradlew.bat" installDebug

# Launch
adb shell am start -n com.example.launchpad/.MainActivity
```

## Context Files

After running `/blueprint`, check `.claude/context/` for:

- **product-vision.md** - App name, problem, solution, unique value
- **technical-spec.md** - Auth, storage, offline, APIs, notifications
- **feature-roadmap.md** - MVP features, phase 2, backlog, out of scope
- **design-guidelines.md** - Style, colors, navigation, inspiration

Always read these files before implementing features to maintain consistency with the user's vision.

## Workflow Summary

1. **User runs `/blueprint`** → Ideation, creates context files
2. **User runs `/forge`** → MVP proposal, builds complete app
3. **Development options:**
   - `/ship <request>` → Autonomous: implement, document, commit
   - `/pilot <request>` → Interactive: build, test, then `/land` to commit
   - `/flight-plan <idea>` → Strategic: plan first, execute later
4. **Use `/run`** → Test changes on emulator
5. **Use `/queue` + `/burn`** → Batch tasks when busy

## Important Notes

- This is a **template project** - it gets transformed into the user's app
- Package name `com.example.launchpad` should be changed during `/forge`
- Always check for connected devices before build/install commands
- Session documentation provides audit trail of all AI decisions
- Product docs in `product-docs/` are optional but helpful

## When Working on This Project

1. **Read context files first** if they exist in `.claude/context/`
2. **Follow existing patterns** in the codebase
3. **Use numbered commits** following the `[NNNN]` convention
4. **Create prompts** for complex multi-step features
5. **Test with `/run`** after making changes
6. **Document sessions** for future reference
